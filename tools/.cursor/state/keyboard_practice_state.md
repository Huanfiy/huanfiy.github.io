# 键盘练习工具状态文档

> 最后更新：2026-02-24  
> 适用范围：`tools/keyboard.html`

## 1. 设计原则

- 刻意练习优先：错误不能被“滑过去”，必须纠正后再推进。
- 反馈闭环清晰：用户要知道目标键是什么、自己按了什么、为什么错。
- 强化应有体感：常错键需要在短期内更高频出现，而不是只做弱概率偏置。
- 节奏自然可持续：强化明显但不过激，避免连续刷同一键造成挫败。
- 数据可解释：统计指标和训练行为要一致，避免“看起来进步但实际逃错”。

## 2. 训练目标

- 提高盲打准确率与按键定位稳定性。
- 缩短“出错 -> 识别错误 -> 纠正”的反馈回路。
- 通过短期强化和长期统计联动，持续压低个人弱键错误率。

## 3. 当前实现（已落地）

### 3.1 模式策略

- 保留模式选择器（基础行/上行/下行/数字行/Shift 组合/功能键/全键混合）。
- 移除独立“智能模式”入口。
- 所有模式统一进入智能采样与强化流程。

### 3.2 推进规则

- 命中目标键：`idx++`，推进到下一键。
- 未命中目标键：不推进，停留当前目标直到打对。
- 结果统计按“尝试次数”记录，避免停留机制导致统计失真。

### 3.3 错误反馈

- 当前目标大键：错误时红色放大并保持，打对后恢复。
- 下方目标键：错误时 `target-wrong` 高亮保持，打对后恢复。
- 下方实际按键：错误时触发 `wrong-press` 跳动提示（帮助识别“我按错了哪个键”）。

### 3.4 强化引擎（混合策略）

- 概率加权：基于长期错误率、短期连错、强化债务进行加权采样。
- 即时回灌：错误键会在未来近距离窗口内被重新插入（受去重和上限控制）。
- 本轮错键加成：同一轮内常错键额外加权，提升“体感强化”。

### 3.5 轮次长度策略

- 基线长度：25 键。
- 出现错误后按阶梯加码，封顶 100 键。
- 无错通关时保持 25 键，不额外拉长。

### 3.6 完成态交互

- 完成弹层显示后，按 `Space` 可直接开始新一轮。

### 3.7 数据持久化

- 使用 `localStorage` 保存按键统计。
- 兼容旧结构（`total/errors`）并扩展短期强化字段。

## 4. 当前关键参数（实现侧）

- 轮次长度：`BASE_SEQ_LEN = 25`，`MAX_SEQ_LEN = 100`
- 权重项：`longTermFactor`、`streakFactor`、`debtFactor`、`streakCap`、`debtCap`
- 即时回灌：`minGap`、`maxGap`、`maxPendingPerKey`、`maxQueueSize`

> 参数为当前手感版本，可继续基于真实练习数据做微调。

## 5. 当前状态结论

- 工具已从“普通模式 + 独立智能模式”转为“全模式默认智能化”。
- 刻意练习链路完整：错键阻塞 -> 错误可视化 -> 短期回灌 -> 长期权重沉淀。
- 目前实现满足“逻辑自然、反馈明确、强化有感”的阶段目标。

## 6. 后续可选优化

- 增加“掌握度”指标（按键级别），补充准确率之外的学习曲线。
- 增加回灌强度档位（标准/增强），兼顾新手与高阶用户手感。
- 在完成弹层展示“本轮回灌命中效果”，提升训练可解释性。
